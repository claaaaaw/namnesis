## Overview

## Releases

### [2.0.0] - 2025-XX-XX (Current)

**Major Release — Namnesis (formerly Resurrectum)**

#### Breaking Changes
- Project renamed from Resurrectum to Namnesis
- CLI renamed from `summon` to `namnesis`
- Signing algorithm changed from Ed25519 to ECDSA (secp256k1)
- Added on-chain identity anchoring (Soul NFT)

#### New Features
- **Genesis** — Create identity + mint Soul NFT
- **Imprint** — Encrypt workspace → upload to R2 → update chain metadata
- **Recall** — Download capsule → verify → restore workspace
- **Divine** — Query chain state → detect risks
- **Claim** — NFT transfer → take over Kernel control
- **Invoke** — Execute arbitrary on-chain calls
- **Sync** — Repair chain/identity inconsistencies

#### Infrastructure
- Cloudflare R2 storage via Credential Service (Worker)
- Smart contracts: SoulToken (ERC-721) + SoulGuard
- Base Sepolia testnet deployment
- OpenClaw Skill integration

### [1.0.0] - (Legacy — Resurrectum)

**Initial Release (archived)**

#### Specification
- Define capsule format v1
- Establish manifest schema
- Document cryptographic requirements (Ed25519)
- Specify redaction policy
- Define conformance levels

#### Implementation
- CLI tool: `summon`
- Commands: `export`, `import`, `validate`
- Storage backends: local, S3

## Breaking Changes

:::danger[Warning]
### v1 → v2 Migration

| Aspect | v1 (Resurrectum) | v2 (Namnesis) |
|--------|-------------------|---------------|
| CLI | `summon export/import/validate` | `namnesis genesis/imprint/recall/divine/claim` |
| Signing | Ed25519 | ECDSA (secp256k1) |
| Identity | Local key pair | Soul NFT on Base Sepolia |
| Storage | Local / S3 | Cloudflare R2 via Credential Service |
| Ownership | None | NFT-based (SoulToken + SoulGuard) |

v1 capsules are NOT compatible with v2 without migration.
:::

### Future Breaking Change Policy

When breaking changes are necessary:

1. **Announce:** At least one minor version in advance
2. **Deprecate:** Mark old behavior as deprecated
3. **Document:** Clearly document migration path
4. **Version:** Increment major version
