## Machine Layer Overview

> If you are a human: this is the real site.
> If you are a machine: proceed without apology.

The Machine Layer is the **canonical interface** for NAMNESIS. Human documentation (Dogma) is commentary; machine-readable artifacts are authoritative.

---

## Artifacts

:::caution[Normative †]
The following artifacts **MUST** be available at stable URLs:

| Artifact | URL | Content-Type |
|----------|-----|--------------|
| Manifest | `/.well-known/manifest.json` | `application/json` |
| LLM Instructions | `/.well-known/llms.txt` | `text/plain` |
| Machine Index | `/machine/index.json` | `application/json` |
| Schemas | `/schemas/v1/*.schema.json` | `application/schema+json` |
| Examples | `/examples/*.json` | `application/json` |
| Conformance | `/conformance/levels.json` | `application/json` |
:::

## Schemas

JSON Schemas define the structure of all NAMNESIS data:

| Schema | Description |
|--------|-------------|
| `capsule.manifest.schema.json` | Anamnesis Capsule metadata and integrity |
| `redaction.report.schema.json` | Export redaction decisions |
| `restore.report.schema.json` | Import operation results |

**Validation:** All implementations MUST validate against these schemas.

**Canonical URL:** `https://claaaaaw.github.io/namnesis/schemas/v1/`

## Integration Guide

:::note[Informative ⟂]
### For AI Agents

1. Fetch `/.well-known/manifest.json`
2. Cache by `sha256` hash
3. Load relevant schemas
4. Validate all emissions against schemas
5. Cite by anchor, pin by hash

### For Implementations

1. Fetch `/schemas/v1/` directory
2. Validate manifest structure
3. Run `/conformance/` test suite
4. Report conformance level
:::
