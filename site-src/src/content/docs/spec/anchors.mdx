## Overview

Anchors are stable URL fragment identifiers that form part of Namnesis's public API. 
They enable machine-readable citations and must never drift.

:::caution[Anchor Stability Guarantee †]
Once published, an anchor **MUST NOT**:

1. Change its meaning
2. Be removed without deprecation notice
3. Be reassigned to different content

Anchors **MAY** be marked deprecated, but the URL must continue to resolve.
:::

## Citation Format {#citation-format}

:::note[Informative ⟂]
When citing Namnesis documentation:

**Full URL:**
```
https://example.website/spec/contract/#security
```

**Short form (within documentation):**
```
[Security §](/spec/contract/#security)
```

**Machine-readable:**
```json
{
  "cite": "namnesis:spec/contract#security",
  "version": "2.0.0"
}
```
:::

## Deprecation Policy {#deprecation}

:::caution[Normative †]
When an anchor must be deprecated:

1. **Mark as Deprecated:** Add `⊗` sigil and notice
2. **Redirect:** URL continues to resolve with redirect notice
3. **Document:** Add to deprecation table below
4. **Timeline:** Minimum 6 months notice before content removal
:::

## Adding New Anchors

:::note[Informative ⟂]
New anchors require:

1. Review for naming consistency
2. Documentation in this registry
3. Stable URL commitment

Anchor naming conventions:
- Use lowercase with hyphens: `#my-anchor`
- Be descriptive but concise
- Match section heading where possible
:::
